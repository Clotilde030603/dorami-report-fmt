# 문체 추출 루프 개발 가이드

이 문서는 일반 사용자 기능이 아니라 `dorami-report-fmt` 스킬 제작 및 개선 방법론을 설명한다.

실제 예시 글은 공개 저장소에 넣지 않는다. 예시 글은 로컬에서만 관리하고, 공개 파일에는 문체 특징을 일반화한 규칙만 반영한다.

## 기본 원칙

- 실제 예시 글, 원본 문서, 변환 문서는 public에 포함하지 않는다.
- 예시 글은 로컬 `private/` 영역에서만 관리한다.
- 예시 문장을 그대로 복사하지 않는다.
- 문체 특징만 일반화해서 `style-guide.md`와 관련 프롬프트에 반영한다.
- 개인정보나 식별 가능한 정보는 제거한다.

## 루프 구조

```text
예시 글 수집
→ private/에서만 관리
→ Markdown 변환 및 익명화
→ 문체 특징 추출
→ style-guide 업데이트
→ 테스트 초안 변환
→ benchmark 평가
→ 규칙 조정
→ 재검증
```

## 문체 추출 항목

- 도입부 방식
- 문단 전개 방식
- 근거 제시 방식
- 결론 표현 방식
- 자주 쓰는 보고서식 표현
- 피하는 표현
- 제목/소제목 구조
- 체크리스트/표 구성 방식
- 문장 길이와 종결어미 패턴

## public 반영 기준

공개 파일에는 실제 문장이 아니라 다음만 반영한다.

- 문체 규칙
- 구조화 원칙
- 금지 표현 기준
- 평가 기준

예시 원문이나 긴 발췌문은 공개 파일에 넣지 않는다.
