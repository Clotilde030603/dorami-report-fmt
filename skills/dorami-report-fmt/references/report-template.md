# 보고서 제목

보고서의 핵심 주제와 범위를 드러내는 제목을 쓴다. 지나치게 포괄적인 제목보다 분석 대상과 관점이 보이는 제목이 좋다.

## 1. 서론

- 보고서의 배경과 목적을 제시한다.
- 문제 상황이나 논의 필요성을 간단히 설명한다.
- 원문에 없는 수치, 사례, 기관명은 추가하지 않는다.

## 2. 본론

### 2.1 핵심 내용

- 사용자가 제공한 주요 사실, 주장, 메모를 정리한다.
- 여러 항목이 있으면 주제별로 묶는다.
- 문장 간 관계가 드러나도록 순서를 조정한다.

### 2.2 분석

- 원인과 결과, 장점과 한계, 문제점과 영향 등을 정리한다.
- 근거가 부족한 부분은 단정하지 않는다.
- 필요한 경우 “확인 필요” 또는 “추가 근거 필요”로 표시한다.

### 2.3 시사점

- 분석 결과가 보고서 주제에 어떤 의미를 갖는지 설명한다.
- 개선 방향이나 검토 과제를 제시할 수 있으나, 원문에 없는 구체적 정책·사례는 만들지 않는다.

## 3. 결론

- 본론의 핵심을 간결하게 정리한다.
- 향후 검토할 사항이나 유의점을 제시한다.
- 새로운 사실을 결론에서 처음 제시하지 않는다.

## 기술·정책 보고서 확장 구조

기술, 정책, 보안 주제의 보고서는 다음 확장 구조를 사용할 수 있다.

### 1. 배경 및 목적

- 보고서 작성 배경, 검토 대상, 목적을 제시한다.
- 넓은 일반론보다 사용자가 제공한 문제 상황을 우선한다.

### 2. 정의 및 현황

- 핵심 용어와 적용 범위를 설명한다.
- 현재 상황이나 관리 기준을 정리한다.

### 3. 문제점 및 위험 요소

- 확인된 문제, 한계, 위험 가능성을 구분한다.
- 근거가 부족한 내용은 단정하지 않는다.

### 4. 개선방안 또는 대응 기준

- 조치 방향, 관리 기준, 점검 항목을 제시한다.
- 체크리스트가 필요한 경우 표로 정리한다.

### 5. 기대효과 및 활용 가능성

- 개선방안이 어떤 효과를 기대할 수 있는지 설명한다.
- 향후 검토 과제나 적용 시 유의점을 함께 제시한다.
