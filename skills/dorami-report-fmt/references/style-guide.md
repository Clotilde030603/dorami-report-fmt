# 도라미 보고서체 스타일 가이드

## 1. 기본 톤

도라미 보고서체는 객관적이고 단정한 한국어 보고서체를 지향한다. 문장은 과장보다 근거, 감상보다 분석, 선언보다 판단 과정을 우선한다.

## 2. 흐름

권장 흐름은 `맥락 → 분석 → 시사점`이다.

1. 맥락: 주제가 왜 다뤄지는지, 어떤 상황에서 논의되는지 설명한다.
2. 분석: 원인, 특징, 문제점, 영향 관계를 정리한다.
3. 시사점: 보고서 관점에서 무엇을 고려해야 하는지 제시한다.

## 3. 문단 구성

- 한 문단은 하나의 핵심 주장이나 설명에 집중한다.
- 첫 문장은 문단의 요지를 제시하고, 다음 문장에서는 근거·조건·한계를 보완한다.
- 원인과 결과가 드러나도록 “이로 인해”, “그 결과”, “이에 따라” 등을 필요한 만큼만 사용한다.

## 4. 문장 길이

- 한 문장은 가능하면 40~70자 안팎으로 유지한다.
- 긴 문장은 원인, 결과, 조건, 시사점을 나누어 쓴다.
- 너무 짧은 문장이 반복되면 보고서 흐름이 끊기므로 연결 관계를 조정한다.

## 5. 제목과 소제목

- 제목은 핵심 주제를 명확히 드러낸다.
- 소제목은 “현황”, “문제점”, “분석”, “개선 방향”, “시사점”처럼 기능이 분명한 표현을 쓴다.
- 과장된 수식어보다 보고서 내용이 예측되는 제목을 우선한다.

## 6. 신중한 표현

근거가 충분하면 단정적으로 쓸 수 있으나, 근거가 부족하면 다음과 같이 조정한다.

- “~이다” → “~로 볼 수 있다”
- “반드시 필요하다” → “검토할 필요가 있다”
- “효과가 크다” → “일정한 효과가 있을 것으로 판단된다”
- “문제를 해결한다” → “문제 완화에 기여할 수 있다”

## 7. 상투적 도입부 피하기

“현대 사회에서 매우 중요한 문제로 대두되고 있다”처럼 넓고 추상적인 도입은 피한다. 대신 보고서 주제와 직접 연결되는 배경, 현황, 문제 상황부터 제시한다.

## 8. 반복 완화

- 같은 종결어미가 반복되면 일부 문장을 명사형, 연결형, 판단형으로 바꾼다.
- “또한”, “따라서”, “이러한”이 연속되면 문단 순서를 재정리하거나 생략한다.
- 같은 구조의 나열은 기능별 소제목이나 표로 전환한다.

## 도라미 보고서체 전개 특징

- 도입부는 넓은 일반론보다 보고서 주제의 배경, 대상, 목적을 바로 제시한다.
- 배경 설명 후 문제점이나 위험 요소를 제시하고, 이후 필요성 또는 관리 기준을 정리하는 흐름이 적합하다.
- 정책, 보안, 기술 주제를 설명할 때는 `정의 → 현황 → 문제점/위험 → 개선방안/관리기준` 순서로 전개한다.
- 시나리오 설명에서는 `환경 → 흐름 → 영향 → 대응 방안` 순서로 정리한다.
- 체크리스트나 표를 활용할 때는 점검 대상, 위험 요소, 확인 기준, 대응 조치를 대응시키는 방식이 적합하다.
- 결론에서는 단순 요약보다 기대효과, 활용 가능성, 향후 검토 과제를 함께 제시한다.
- 문장 길이는 지나치게 짧게 끊기보다 원인과 결과를 한 문단 안에서 연결하되, 복잡한 내용은 소제목·목록·표로 분리한다.
- 연결 표현은 필요한 곳에만 사용하고, 같은 표현이 반복되면 문단 순서나 문장 구조를 조정한다.

## 표·체크리스트·시나리오 작성 방식

- 표는 설명을 대체하는 장식 요소가 아니라, 위험 요소와 점검 기준을 빠르게 대응시키는 도구로 사용한다.
- 체크리스트는 항목명만 나열하지 말고 점검 목적, 확인 기준, 조치 방향을 구분한다.
- 시나리오는 배경 상황을 길게 늘리지 말고, 행위 흐름과 영향, 대응 방안을 단계적으로 제시한다.
- 기술·정책 보고서에서는 용어 정의를 먼저 제시한 뒤, 관리 필요성과 적용 기준을 설명한다.


## 핵심 전개 패턴

- 배경 설명 후 문제점을 제시하고 필요성을 정리한다.
- 정책·보안·기술 주제는 `정의 → 현황 → 문제점/위험 → 개선방안/관리기준` 순서로 전개한다.
- 시나리오형 내용은 `환경 → 흐름 → 영향 → 대응 방안` 순서로 구성한다.
- 체크리스트형 내용은 점검 대상, 위험 요소, 확인 기준, 대응 조치를 구분한다.
- 결론은 핵심 요약, 기대효과, 활용 가능성, 향후 검토 과제를 균형 있게 담는다.
- 근거가 부족한 내용은 단정하지 않고 “~로 볼 수 있다”, “~로 판단된다”, “~할 필요가 있다”처럼 표현한다.
- 같은 연결 표현과 종결어미가 반복되지 않도록 한다.
- 과장 표현보다 근거 중심 문장을 사용한다.
