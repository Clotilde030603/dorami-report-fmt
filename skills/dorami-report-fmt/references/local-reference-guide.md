# 개발자용 로컬 참고자료 관리 가이드

이 문서는 스킬 유지보수자가 개인 보고서 자료를 로컬에서만 다룰 때의 안전 기준을 정리한다. 일반 사용자는 이 문서를 사용할 필요가 없다.

## 기본 원칙

- 개인 보고서 예시는 공개 저장소에 올리지 않는다.
- 로컬 자료는 문체와 구조를 점검하기 위한 참고용이며, 문장 복사용이 아니다.
- 이름, 학교, 학번, 이메일, 전화번호, 생년월일, 팀명 등 식별 정보는 제거한다.
- 공개 가능한 자료인지 유지보수자가 직접 확인한다.
- 공개 배포물에는 실제 예시 파일이 아니라 일반화된 문체 규칙만 남긴다.

## 로컬 자료 보관

로컬 자료가 필요한 경우 현재 프로젝트 폴더 안의 ignored 경로만 사용한다.

```text
private/
```

`private/`는 `.gitignore`에 포함되어야 하며, GitHub에 업로드하지 않는다.

## 공개 파일 반영 기준

공개 파일에는 다음만 반영한다.

- 문체 규칙
- 보고서 구조 원칙
- 금지 표현 가이드
- 검증 스크립트와 설치 방법

공개 파일에는 다음을 반영하지 않는다.

- 실제 보고서 원문
- 변환된 예시 문서
- 익명화된 예시 문서
- 개인을 식별할 수 있는 정보
