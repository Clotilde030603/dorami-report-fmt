# AI식 표현 완화 가이드

이 문서는 `dorami-report-fmt`가 도라미 보고서체를 적용할 때 참고하는 내부 작성 규칙이다.

별도의 외부 도구 실행이나 후처리 단계를 의미하지 않는다.

목적은 원문의 의미를 유지하면서 상투 표현, 번역투, 반복 문장 구조를 줄이고 보고서 문체를 자연스럽게 만드는 것이다.

## 1. 상투적 도입부 완화

- 넓은 사회 일반론으로 시작하지 않는다.
- 보고서의 대상, 배경, 목적을 바로 제시한다.
- “현대 사회에서 매우 중요한 문제로 대두되고 있다”처럼 포괄적인 도입은 피한다.

## 2. 번역투 완화

- “~를 통해”가 반복되면 “~로”, “~을 활용해”, “~에 따라” 등 문맥에 맞게 바꾼다.
- “~에 있어서”는 “~에서”, “~의”, “~와 관련해”로 줄일 수 있다.
- “~에 의해”가 반복되면 행위 주체를 드러내는 능동형을 검토한다.

## 3. 과도한 접속사 완화

- “또한”, “따라서”, “이러한”, “나아가”가 문두에 반복되면 일부를 생략한다.
- 접속사가 없어도 문맥이 이어지면 제거한다.
- 논리 전환이 필요한 곳에만 접속 표현을 남긴다.

## 4. 기계적인 나열 완화

- “첫째, 둘째, 셋째” 구조가 반복되면 내용별 소제목으로 바꾼다.
- 중요도나 원인-결과 관계가 있으면 단순 나열보다 문단 흐름으로 정리한다.
- 체크리스트가 적합한 내용은 항목, 기준, 조치 방향을 나누어 쓴다.

## 5. 피동태 남용 완화

- “~되어진다”, “~되어 있다”가 반복되면 능동형이나 간결한 피동형으로 바꾼다.
- 관리 주체나 조치 주체가 중요한 문장은 주체를 드러낸다.

## 6. 불필요한 영어 병기 완화

- 이미 널리 쓰이는 한국어 용어가 있으면 영어 병기를 줄인다.
- 전문용어가 필요한 경우 최초 1회만 병기하고 이후에는 한국어 표현을 사용한다.

## 7. 반복 종결어미 완화

- “~이다”가 반복되면 일부 문장은 “~로 볼 수 있다”, “~로 판단된다”, “~한 점에서 의미가 있다”로 조정한다.
- 단, 완곡 표현을 과도하게 겹치지 않는다.
- 같은 문장 길이와 같은 구조가 반복되면 문단 흐름을 재구성한다.

## 8. 보고서체를 해치지 않는 자연스러운 대체 표현

- “좋다” → “적절하다”, “효과적일 수 있다”
- “문제가 많다” → “여러 한계가 확인된다”
- “꼭 해야 한다” → “검토할 필요가 있다”
- “대단히 중요하다” → “중요한 검토 요소다”
- “많은 도움이 된다” → “실무 적용 가능성을 높인다”

## 9. 적용 원칙

- 원문 의미와 사실관계를 우선 보존한다.
- 과한 수식어보다 근거 중심 문장을 사용한다.
- 보고서체의 객관성과 신중함을 해치지 않는 범위에서만 자연스럽게 수정한다.
